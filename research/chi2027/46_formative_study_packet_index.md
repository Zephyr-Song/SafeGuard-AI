## Before First Pilot Session

Complete:

- [ ] supervisor approves problem framing;
- [ ] supervisor approves participant type;
- [ ] supervisor approves whether formal ethics/IRB review is needed before pilot;
- [ ] run one internal dry run;
- [ ] test export and file naming;
- [ ] prepare participant folder structure;
- [ ] decide whether sessions are audio recorded, screen recorded, or note-only;
- [ ] decide whether to use English only, Chinese only, or bilingual materials.
- [ ] fill `47_recruitment_tracking_table.csv` with candidate participants.

## First Internal Dry Run

Suggested dry run:

- one labmate or self-run session;
- use participant study mode;
- complete initial plan, rehearsal, reflection, revised plan;
- export session;
- code the plan pair with `45_coding_protocol_v1.md`;
- update confusing instructions before recruiting real participants.

Use:

## Supervisor Decisions Needed

Ask supervisor:

1. Is the problem framing narrow and original enough for CHI?
2. Should the first formative participants be research students/design researchers, or should we include ethics reviewers/community-engaged researchers?
3. Is online fraud prevention the main domain or an initial case of sensitive digital safety action research?
4. Is 6-8 participants enough for the formative contribution?
5. What ethics/approval path is required before recruiting?

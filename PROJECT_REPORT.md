# SafeGuard AI 项目报告
## 基于Fraud-R1改进的安全教育与AI伦理学习平台

---

## 1. 项目概述

### 1.1 项目背景
本项目基于Fraud-R1（一个多轮次LLM反欺诈基准测试框架）进行改进，将其从**攻击测试工具**转变为**安全教育与AI伦理学习平台**。

### 1.2 项目目标
- **教育目标**：帮助用户识别和防范网络诈骗
- **伦理目标**：理解AI决策的透明度、偏见与公平性
- **技术目标**：实现一个安全、可解释、负责任的AI教育系统

### 1.3 核心创新
1. **模式转换**：从攻击测试 → 教育学习
2. **透明度仪表板**：展示AI决策过程
3. **偏见检测**：识别和分析AI系统中的潜在偏见
4. **反事实推理**：探索"如果...会怎样"的场景
5. **安全守护**：创伤知情的安全检查机制

## 2. 系统架构

### 2.1 整体架构
```
┌─────────────────────────────────────────────────────────────┐
│                    SafeGuard AI Platform                     │
├─────────────────────────────────────────────────────────────┤
│  Frontend (HTML/CSS/JS)                                      │
│  ├── 互动学习界面                                            │
│  ├── 透明度仪表板                                            │
│  └── 学习进度追踪                                            │
├─────────────────────────────────────────────────────────────┤
│  Backend (Flask)                                             │
│  ├── Education Engine (教育引擎)                             │
│  ├── Scenario Simulator (场景模拟器)                         │
│  ├── Transparency Dashboard (透明度仪表板)                   │
│  ├── Safety Guardian (安全守护)                              │
│  └── Learning Tracker (学习追踪)                             │
├─────────────────────────────────────────────────────────────┤
│  Data Layer                                                  │
│  ├── 诈骗场景数据库                                          │
│  ├── 学习进度存储                                            │
│  └── 决策历史记录                                            │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 核心模块

#### 2.2.1 教育引擎 (EducationEngine)
- **功能**：生成个性化安全教育内容和学习路径
- **特点**：
  - 多阶段学习（意识建立→知识学习→实践练习→反思总结）
  - 自适应难度调整
  - 实时反馈与解释

#### 2.2.2 场景模拟器 (ScenarioSimulator)
- **功能**：创建和管理互动式学习场景
- **特点**：
  - 5种诈骗类型模拟
  - 多轮对话支持
  - 渐进式难度递增

#### 2.2.3 透明度仪表板 (TransparencyDashboard)
- **功能**：展示AI决策过程
- **特点**：
  - 决策因素可视化
  - 偏见检测与报告
  - 反事实场景推理

#### 2.2.4 安全守护 (SafetyGuardian)
- **功能**：内容distress检测和安全检查
- **特点**：
  - 情绪状态监控
  - 创伤知情安全提示
  - 分级干预机制

#### 2.2.5 学习追踪 (LearningTracker)
- **功能**：追踪学习进度和生成报告
- **特点**：
  - 成就系统
  - 进度可视化
  - 个性化建议

## 3. 功能演示

### 3.1 互动学习流程

#### 步骤1：选择学习场景
用户可以从5种诈骗类型中选择：
- 🎭 虚假服务
- 👤 身份冒充
- 🎣 钓鱼诈骗
- 💼 虚假招聘
- 💕 网恋诈骗

#### 步骤2：场景模拟
系统展示模拟的诈骗场景，例如：
```
📱 来自: 李教练
您好，这里是环城驾校，我们可以代办B1驾照，无需考试，
只需支付800元定金和7500元考试费。
请转账至账户：6228 4801 2665 3397 775
```

#### 步骤3：用户响应与分析
用户输入对场景的判断，系统分析：
- 识别的风险点
- 遗漏的风险点
- 安全得分
- 个性化反馈

### 3.2 透明度分析示例

#### 决策因素分析
```
风险指标分析 (权重 35%)
├── 检测到风险关键词: 承诺无需考试, 要求预付费用
├── 置信度: 85%
└── 证据: 个人账户收款

紧急性分析 (权重 25%)
├── 检测到紧急性语言
├── 置信度: 75%
└── 证据: 立即, 马上

发送者验证 (权重 25%)
├── 未经验证的联系方式
├── 置信度: 80%
└── 证据: 非官方账户
```

#### 反事实推理
```
场景: 如果发送者可以通过官方渠道验证
├── 改变因素: 发送者验证状态, 联系方式可信度
├── 预测结果: 风险等级将显著降低
└── 置信度: 85%

场景: 如果是面对面交流
├── 改变因素: 交流方式, 身份验证难度
├── 预测结果: 风险识别将更加容易
└── 置信度: 80%
```

### 3.3 偏见检测示例

#### 人口统计偏见
- **检测**: 内容是否针对特定群体（老年人、学生、家庭主妇）
- **结果**: 诈骗者常针对特定群体设计话术
- **缓解**: 这反映了现实中的诈骗策略，而非AI偏见

#### 语言偏见
- **检测**: 中英文使用比例
- **结果**: 系统支持双语分析
- **缓解**: 语言偏好不影响风险评估准确性

#### 场景偏见
- **检测**: 场景类型分布
- **结果**: 金融、情感、紧急场景分布
- **缓解**: 建议接触多种类型的诈骗场景
